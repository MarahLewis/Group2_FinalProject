This project is a restaurant ordering and processing system. It is meant to manage a restaurant's kitchen and will simulate how a real restaurant will handle orders from start to finish. In many restaurants, orders can get mixed up, delayed, or placed incorrectly, especially if there is no system in place. This project will solve that problem by organizing menu items, customers, employees, and orders to ensure clean workflow. Orders will be placed, then added to a queue, prepared in order, and then include a receipt that has totals and discounts. Overall, the goal of this project is to show how OOP can be used to create a structured system that will improve the organization and minimize mistakes in a restaurant's environment.
